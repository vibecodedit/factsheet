# Contributing to FACT SHEET

Thank you for your interest in improving FACT SHEET! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

If you find a problem with the template or have a suggestion:

1. Check if the issue already exists in the [Issues](../../issues) section
2. If not, create a new issue with:
   - A clear, descriptive title
   - Detailed description of the problem or suggestion
   - Examples or use cases if applicable

### Suggesting Template Improvements

Have an idea for a new section or improvement to the template?

1. Open an issue describing your proposed change
2. Explain the use case and why it would be valuable
3. Provide an example of what the new section would look like
4. Wait for discussion and feedback before starting work

### Submitting Changes

1. **Fork the repository** to your own GitHub account

2. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**:
   - Edit the relevant files
   - Keep changes focused on a single improvement
   - Maintain the existing formatting and style

4. **Test your changes**:
   - Ensure the Markdown renders correctly
   - Verify all links work
   - Check that examples are clear and accurate

5. **Commit your changes**:
   ```bash
   git add .
   git commit -m "Brief description of your changes"
   ```

6. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**:
   - Go to the original FACT SHEET repository
   - Click "New Pull Request"
   - Select your branch
   - Provide a clear description of your changes
   - Reference any related issues

## Guidelines

### Template Changes

- Keep sections clear and easy to understand
- Use consistent formatting throughout
- Provide placeholder text that guides users
- Include helpful examples where appropriate
- Maintain the "single Markdown file" philosophy

### Documentation Changes

- Use clear, concise language
- Check spelling and grammar
- Ensure links are valid
- Keep the tone professional but approachable

### Example Files

- Examples should be realistic but generic
- Don't include real IP addresses, credentials, or proprietary information
- Make examples comprehensive enough to demonstrate value

## What We're Looking For

We're especially interested in contributions that:

- Add valuable new template sections
- Improve clarity of existing sections
- Fix errors or broken links
- Enhance examples with real-world scenarios
- Improve documentation and usage instructions

## What We're Not Looking For

- Changes that make the template overly complex
- Multiple files or tooling dependencies
- Template variations for specific technologies (keep it generic)
- Automated processing or parsing tools (keep it simple)

## Code of Conduct

- Be respectful and professional
- Welcome newcomers and different perspectives
- Focus on constructive feedback
- Keep discussions on-topic

## Questions?

If you have questions about contributing, feel free to:
- Open an issue with the "question" label
- Start a discussion in the Discussions tab (if enabled)
